## 🚀 快速开始

### 硬件组装（30-60 分钟）

1. **准备组件**：按照 `hardware_design.md` 中的清单准备所有组件
2. **焊接电路**：按照接线图焊接 ESP32-C3、光耦、电阻、电池座
3. **连接车钥匙**：用细导线连接光耦输出端到原车钥匙的按键触点
4. **测试电源**：用万用表检查电源和地之间是否存在短路

### 固件上传（10-15 分钟）

1. **安装 Arduino IDE**：从 [arduino.cc](https://www.arduino.cc/en/software) 下载
2. **添加 ESP32 支持**：在首选项中添加 ESP32 开发板 URL
3. **选择开发板**：工具 → 开发板 → ESP32C3 Dev Module
4. **上传固件**：打开 `firmware.ino`，点击上传按钮

### App 部署（5-10 分钟）

```bash
# 进入项目目录
cd /home/ubuntu/car_key_ble_app

# 安装依赖
pnpm install

# 启动开发服务器
pnpm dev

# 或生产构建
pnpm build
pnpm start
```

访问 `http://localhost:3000` 打开应用。

### 首次使用

1. 打开手机 App

## 🔧 技术栈

### 硬件

- **主控**：ESP32-C3（超低功耗 BLE SoC）
- **隔离**：4 路光耦隔离模块（TLP521-4）
- **电源**：CR2032 纽扣电池（220 mAh）
- **驱动**：430Ω 限流电阻

### 固件

- **语言**：C/C++（Arduino 框架）
- **BLE 栈**：ESP32 原生 BLE 库
- **功耗管理**：Light-sleep + Deep-sleep

### 手机 App

- **框架**：React 19 + TypeScript
- **UI**：shadcn/ui + Tailwind CSS 4
- **BLE 通信**：Web Bluetooth API
- **路由**：Wouter

## 📊 功耗分析

| 模式 | 电流 | 占比 | 续航贡献 |
|------|------|------|--------|
| Deep-sleep | 5 μA | 99.9% | 4.995 μA |
| Light-sleep (BLE 广播) | 200 μA | 0.09% | 0.18 μA |
| Active (执行命令) | 80 mA | 0.01% | 0.8 μA |
| **平均总计** | - | - | **~6 μA** |

**续航计算**：
- CR2032 容量：220 mAh
- 平均电流：6 μA
- 续航时间：220 mAh ÷ 6 μA ≈ **36,667 小时** ≈ **4.2 年**

实际续航会因使用频率而有所不同，但即使在频繁使用的情况下，也能轻松超过半年。

## 🔐 安全机制

### 1. BLE 加密连接

- 使用 BLE Secure Connections（ECDH 密钥交换）
- 支持 MITM（中间人攻击）防护
- 所有通信数据都经过加密

### 2. PIN 码认证

- 首次配对时需要输入 6 位 PIN 码
- PIN 码通过加密通道发送
- 支持配对绑定，防止重复配对

### 3. 距离限制

- BLE 信号强度限制在 3-5 米范围内
- 超出范围自动断开连接
- 物理距离提供额外的安全保障

### 4. 光耦隔离

- 完全电气隔离 ESP32-C3 与车钥匙电路
- 防止电流倒灌和电压不匹配
- 保护原车钥匙电路不受损伤

## 🛠️ 故障排查

### 常见问题

| 问题 | 解决方案 |
|------|--------|
| 找不到设备 | 检查 ESP32-C3 是否上电，确保在 3-5 米范围内 |
| 连接失败 | 检查 PIN 码是否正确，尝试重新配对 |
| 命令无响应 | 检查光耦接线是否正确，确认 GPIO 输出正常 |
| 电池续航短 | 检查固件是否正确配置了低功耗模式 |
| 信号弱 | 靠近设备，检查是否有金属或水的干扰 |

### 调试方法

1. **串口监视器**：连接 ESP32-C3 到电脑，打开 Arduino IDE 的串口监视器查看日志
2. **BLE 扫描器**：使用 nRF Connect 等 BLE 扫描工具检查设备广播
3. **万用表测试**：检查电源、地、GPIO 输出电压
