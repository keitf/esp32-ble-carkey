# 车钥匙蓝牙外挂 - 完整实施指南

## 项目概述

本项目设计了一套基于 **ESP32-C3 + 4路光耦隔离模块 + CR2032 纽扣电池** 的车钥匙蓝牙近场控制方案。该方案不需要拆卸原车钥匙内部电路，也不需要在车上安装任何设备，仅通过蓝牙通信实现手机近场控制（3-5 米范围内）。

### 核心特性

| 特性 | 说明 |
|------|------|
| **功耗** | 超低功耗设计，CR2032 电池续航半年以上 |
| **安全性** | BLE 加密连接 + 6位 PIN 码验证 + 隐藏广播 |
| **距离** | 3-5 米蓝牙近场控制，超出范围无法连接 |
| **隔离** | 光耦隔离保护原车钥匙电路 |
| **功能** | 开锁、关锁、开后备箱 |
| **无依赖** | 无网络、无 WiFi、无车载设备 |

## 硬件部分

### 1. 核心组件清单

| 组件 | 型号/规格 | 数量 | 用途 |
|------|---------|------|------|
| ESP32-C3 模组 | ESP32-C3-MINI-1 或类似 | 1 | 主控芯片，BLE 通信 |
| 4 路光耦隔离模块 | TLP521-4 或类似 | 1 | 隔离 ESP32-C3 与车钥匙电路 |
| CR2032 纽扣电池 | 3V, 220mAh | 1 | 独立电源 |
| 限流电阻 | 430Ω 1/4W | 4 | 光耦 LED 驱动 |
| 电池座 | CR2032 座子 | 1 | 电池安装 |
| 焊接材料 | 锡焊、焊接膏 | - | PCB 焊接 |

### 2. 接线原理

#### 2.1 供电部分

```
CR2032 电池
  ├─ 正极 (+) → ESP32-C3 VCC (3.3V)
  └─ 负极 (-) → ESP32-C3 GND
```

**注意**：CR2032 电池可直接为 ESP32-C3 供电，无需额外的升压或降压电路。

#### 2.2 光耦驱动部分

```
ESP32-C3 GPIO1 → 430Ω 电阻 → 光耦1 LED+ (VCC)
                                    ↓
                              光耦1 LED- (GND)

光耦1 输出 (C) → 车钥匙按键1 触点 A
光耦1 输出 (E) → 车钥匙按键1 触点 B
```

**四个通道的接线方式相同**：

| GPIO 引脚 | 光耦通道 | 功能 | 车钥匙按键 |
|---------|--------|------|---------|
| GPIO1 | CH1 | 开锁 | 开锁按键 |
| GPIO2 | CH2 | 关锁 | 关锁按键 |
| GPIO3 | CH3 | 开后备箱 | 后备箱按键 |
| GPIO4 | CH4 | 预留 | 预留功能 |

#### 2.3 光耦输出端连接

光耦的输出端（光敏三极管）需要连接到原车钥匙 PCB 上对应按键的两个触点。当光耦导通时，会模拟按键按下时的短路状态。

**连接步骤**：

1. 拆开原车钥匙外壳，暴露 PCB
2. 用万用表或示波器识别各按键的触点位置
3. 用细导线（如 30AWG 铜线）焊接光耦输出端到按键触点
4. 确保焊接点牢固且不会短路到其他电路

### 3. 功耗估算与电池续航

**CR2032 电池规格**：
- 容量：220 mAh
- 额定电压：3V
- 最大脉冲电流：100 mA

**续航目标**：半年以上（约 4380 小时）

**平均电流预算**：
$$I_{avg} = \frac{220 \text{ mAh}}{4380 \text{ h}} \approx 50 \mu A$$

**功耗模式分析**：

| 模式 | 电流 | 占比 | 贡献 |
|------|------|------|------|
| Deep-sleep | 5 μA | 99.9% | 4.995 μA |
| Light-sleep (BLE 广播) | 200 μA | 0.09% | 0.18 μA |
| Active (执行命令) | 80 mA | 0.01% | 0.8 μA |
| **总计** | - | - | **~5.975 μA** |

该估算表明，通过合理的功耗管理，可以轻松实现半年以上的续航。实际续航时间可能更长，取决于 BLE 广播间隔和连接频率。

### 4. 焊接与组装

#### 4.1 焊接顺序

1. **焊接电池座**：在 PCB 上焊接 CR2032 电池座，确保正负极标记正确
2. **焊接 ESP32-C3 模组**：使用细焊丝或 BGA 焊接工艺焊接 ESP32-C3 模组
3. **焊接限流电阻**：在 ESP32-C3 的 GPIO 引脚和光耦 LED 输入端之间焊接 430Ω 电阻
4. **焊接光耦模块**：焊接 4 路光耦隔离模块到 PCB
5. **焊接输出导线**：用细导线连接光耦输出端到原车钥匙的按键触点

#### 4.2 焊接工具

- 烙铁（25-40W）
- 焊锡（无铅或有铅）
- 焊接膏（可选，用于 SMD 焊接）
- 热风枪（可选，用于 BGA 焊接）
- 放大镜或显微镜
- 镊子和焊接辅助工具

#### 4.3 焊接注意事项

- 避免过热，防止 ESP32-C3 模组损坏
- 确保焊点饱满且无冷焊
- 焊接完成后用酒精清洁 PCB，去除焊接残留物
- 用万用表检查电源和地之间是否存在短路

## 固件部分

### 1. 开发环境配置

#### 1.1 Arduino IDE 配置

1. 下载并安装 [Arduino IDE](https://www.arduino.cc/en/software)
2. 在 Arduino IDE 中添加 ESP32 开发板支持：
   - 打开 **文件 → 首选项**
   - 在 **附加开发板管理器网址** 中添加：`https://raw.githubusercontent.com/espressif/arduino-esp32/gh-pages/package_esp32_index.json`
   - 打开 **工具 → 开发板 → 开发板管理器**
   - 搜索 "ESP32" 并安装 Espressif Systems 提供的包

#### 1.2 选择开发板

- 打开 **工具 → 开发板** 并选择 **ESP32C3 Dev Module**
- 设置 **工具 → 端口** 为 ESP32-C3 的串口

### 2. 固件代码

完整的固件代码已在 `firmware.ino` 文件中提供。该固件实现了以下功能：

- **BLE 服务和特征值**：定义了一个 BLE 服务和特征值，用于接收来自手机的命令
- **安全配对**：配置了 BLE 安全连接和固定 PIN 码（123456）
- **GPIO 控制**：通过 GPIO 引脚驱动光耦，模拟按键按下
- **低功耗管理**：配置了自动 Light-sleep 模式以降低功耗
- **命令处理**：解析来自手机的 "UNLOCK"、"LOCK"、"TRUNK" 命令

### 3. 固件编译与上传

1. 在 Arduino IDE 中打开 `firmware.ino` 文件
2. 点击 **验证** 按钮编译代码
3. 连接 ESP32-C3 到电脑的 USB 端口
4. 点击 **上传** 按钮上传固件到 ESP32-C3

### 4. 固件参数调整

根据实际需求，可以调整以下参数：

```cpp
#define DEVICE_NAME "CarKey_BLE"      // BLE 设备名称
#define PASSKEY 123456                 // 6位 PIN 码

#define PIN_UNLOCK 1                   // 开锁 GPIO 引脚
#define PIN_LOCK 2                     // 关锁 GPIO 引脚
#define PIN_TRUNK 3                    // 后备箱 GPIO 引脚
#define PIN_AUX 4                      // 预留 GPIO 引脚

// BLE UUIDs (建议使用随机生成的 UUID)
#define SERVICE_UUID "4fafc201-1fb5-459e-8fcc-c5c9c331914b"
#define CHARACTERISTIC_UUID "beb5483e-36e1-4688-b7f5-ea07361b26a8"
```

### 5. 低功耗优化

为了实现半年以上的续航，固件已配置了以下低功耗策略：

- **自动 Light-sleep**：当没有活动时，ESP32-C3 会自动进入 Light-sleep 模式
- **长广播间隔**：BLE 广播间隔设置为 1-2 秒，平衡可发现性和功耗
- **快速断开**：连接完成后立即断开，返回 Light-sleep 模式
- **Deep-sleep 唤醒**：可通过外部中断或定时器唤醒 Deep-sleep 模式

## 使用流程

### 1. 首次使用

1. **组装硬件**：按照硬件接线部分组装 ESP32-C3 + 光耦 + CR2032 电池
2. **上传固件**：使用 Arduino IDE 上传固件到 ESP32-C3
3. **打开 App**：在支持 Web Bluetooth 的浏览器中打开手机 App
4. **扫描设备**：点击 "扫描设备" 按钮，搜索 "CarKey_BLE"
5. **输入 PIN 码**：首次连接时输入 PIN 码 (默认为 123456)
6. **配对成功**：设备连接后，可以开始使用

### 2. 日常使用

1. **靠近车钥匙**：将手机靠近车钥匙（3-5 米范围内）
2. **打开 App**：打开手机 App，自动搜索并连接设备
3. **执行操作**：点击 "开锁"、"关锁" 或 "开后备箱" 按钮
4. **操作完成**：App 会显示操作结果，设备自动断开连接

### 3. 故障排查

| 问题 | 解决方案 |
|------|--------|
| 找不到设备 | 检查 ESP32-C3 是否上电，确保在 3-5 米范围内 |
| 连接失败 | 检查 PIN 码是否正确，尝试重新配对 |
| 命令无响应 | 检查光耦接线是否正确，确认 GPIO 输出正常 |
| 电池续航短 | 检查固件是否正确配置了低功耗模式 |
| 信号弱 | 靠近设备，检查是否有金属或水的干扰 |

## 安全建议

### 1. PIN 码管理

- **更改默认 PIN**：在固件中修改 `PASSKEY` 值为自定义的 6 位数字
- **不要共享 PIN**：PIN 码应该只有你知道
- **定期更换 PIN**：建议每 3 个月更换一次 PIN 码

### 2. BLE 安全

- **启用加密**：固件已配置了 BLE 加密连接，确保通信安全
- **隐藏广播**：可以修改固件，使设备仅在特定条件下进行广播
- **距离限制**：蓝牙信号强度已限制在 3-5 米范围内，超出范围无法连接

### 3. 物理安全

- **保护设备**：将 ESP32-C3 + 光耦模块放在安全的地方，防止被盗或损坏
- **定期检查**：定期检查焊接点是否牢固，电池是否需要更换
- **备用方案**：始终保留原车钥匙作为备用

## 故障排查与调试

### 1. 串口监视器调试

连接 ESP32-C3 到电脑，打开 Arduino IDE 的串口监视器（波特率 115200）查看调试信息：

```
BLE Server is ready. Waiting for connections...
Device connected
Received Value: UNLOCK
Triggering PIN: 1
Device disconnected
Restarting advertising
```

### 2. 常见错误

| 错误信息 | 原因 | 解决方案 |
|--------|------|--------|
| `Authentication Failed` | PIN 码错误 | 检查固件中的 PASSKEY 设置 |
| `Connection timeout` | 设备超出范围 | 靠近设备，检查 BLE 广播 |
| `GPIO write failed` | GPIO 配置错误 | 检查 GPIO 引脚定义 |
| `Battery low` | 电池电压过低 | 更换新的 CR2032 电池 |

### 3. 性能优化

- **减少广播间隔**：如果需要更快的响应速度，可以减少广播间隔（但会增加功耗）
- **增加广播功率**：如果需要更远的距离，可以增加 BLE 广播功率（但会增加功耗）
- **优化连接参数**：调整 BLE 连接参数以平衡延迟和功耗
