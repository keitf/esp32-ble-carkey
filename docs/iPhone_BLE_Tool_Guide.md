## 方案 A：使用 LightBlue（推荐）

### 第 1 步：下载和安装

1. 打开 iPhone 上的 **App Store**
2. 搜索 **"LightBlue"**（开发者：Punchthrough Design）
3. 点击 **获取** 下载安装
4. 打开 App

### 第 2 步：扫描设备

1. 打开 LightBlue App
2. 点击屏幕上的 **"Scan"** 按钮
3. 等待 App 扫描附近的 BLE 设备（通常需要 3-5 秒）
4. 在设备列表中找到 **"CarKey_BLE"**

**如果看不到设备**：
- 确保 ESP32-C3 已上电（检查 CR2032 电池）
- 确保 iPhone 与设备距离在 3-5 米以内
- 检查 iPhone 的蓝牙是否已打开（设置 → 蓝牙）
- 尝试重启 ESP32-C3 或 iPhone 的蓝牙

### 第 3 步：连接设备

1. 在设备列表中点击 **"CarKey_BLE"**
2. App 会显示连接进度
3. 系统可能会弹出 PIN 码输入框
4. 输入 PIN 码：**123456**（或你在固件中设置的 PIN 码）
5. 等待连接成功

**连接成功的标志**：
- 设备名称旁边显示 ✓ 符号
- App 显示 "Connected" 状态
- 可以看到设备的服务列表

### 第 4 步：找到控制特征值

1. 连接成功后，向下滚动查看 **Services** 列表
2. 找到 Service UUID：**4fafc201-1fb5-459e-8fcc-c5c9c331914b**
3. 点击展开该服务
4. 在 Characteristics 中找到：**beb5483e-36e1-4688-b7f5-ea07361b26a8**
5. 点击该特征值展开

### 第 5 步：发送控制命令

在特征值详情页面中，找到 **"Write new value"** 输入框，按以下步骤操作：

#### 开锁
1. 在输入框中输入：**UNLOCK**
2. 点击 **"Write"** 按钮
3. 等待 1-2 秒，车钥匙会执行开锁动作
4. App 会显示 "Write successful" 提示

#### 关锁
1. 在输入框中输入：**LOCK**
2. 点击 **"Write"** 按钮
3. 等待 1-2 秒，车钥匙会执行关锁动作

#### 开后备箱
1. 在输入框中输入：**TRUNK**
2. 点击 **"Write"** 按钮
3. 等待 1-2 秒，车钥匙会执行开后备箱动作

### 第 6 步：断开连接

操作完成后，点击 **"Disconnect"** 按钮断开连接。设备会自动返回低功耗模式。

## 方案 B：使用 nRF Connect

### 第 1 步：下载和安装

1. 打开 iPhone 上的 **App Store**
2. 搜索 **"nRF Connect"**（开发者：Nordic Semiconductor）
3. 点击 **获取** 下载安装
4. 打开 App

### 第 2 步：扫描设备

1. 打开 nRF Connect App
2. 点击 **"Scan"** 按钮开始扫描
3. 在扫描结果中找到 **"CarKey_BLE"**

### 第 3 步：连接设备

1. 点击 **"CarKey_BLE"** 设备
2. 点击 **"Connect"** 按钮
3. 如果提示输入 PIN 码，输入：**123456**
4. 等待连接成功

### 第 4 步：找到控制特征值

1. 连接成功后，点击 Service UUID：**4fafc201-1fb5-459e-8fcc-c5c9c331914b**
2. 在 Characteristics 中找到：**beb5483e-36e1-4688-b7f5-ea07361b26a8**
3. 点击该特征值

### 第 5 步：发送控制命令

在特征值详情中：

1. 点击 **"Write"** 按钮
2. 在弹出的输入框中输入命令：
   - **UNLOCK**（开锁）
   - **LOCK**（关锁）
   - **TRUNK**（开后备箱）
3. 点击 **"Send"** 或 **"Write"** 按钮
4. 等待命令执行

## 📋 快速参考表

| 操作 | 命令 | 预期结果 |
|------|------|--------|
| 开锁 | `UNLOCK` | 车钥匙执行开锁动作（GPIO1 输出 500ms 高电平） |
| 关锁 | `LOCK` | 车钥匙执行关锁动作（GPIO2 输出 500ms 高电平） |
| 开后备箱 | `TRUNK` | 车钥匙执行开后备箱动作（GPIO3 输出 500ms 高电平） |

## 🔐 安全提示

1. **PIN 码保护**：首次连接需要输入 PIN 码，防止未授权连接
2. **距离限制**：蓝牙信号限制在 3-5 米范围内，超出范围无法连接
3. **加密通信**：所有命令都通过 BLE 加密通道传输
4. **自动断开**：操作完成后应立即断开连接，节省电池

## ⚠️ 常见问题

### Q1：为什么找不到 "CarKey_BLE" 设备？

**原因和解决方案**：

| 原因 | 解决方案 |
|------|--------|
| ESP32-C3 未上电 | 检查 CR2032 电池是否安装正确，用万用表测量电压 |
| 距离太远 | 靠近设备，确保在 3-5 米范围内 |
| 蓝牙未打开 | 打开 iPhone 设置 → 蓝牙 |
| 设备在广播休眠 | 等待 1-2 秒，设备会定期广播 |
| 其他 App 占用连接 | 关闭其他 BLE App，重启蓝牙 |

### Q2：连接时提示 "Authentication Failed"？

**解决方案**：
- 检查输入的 PIN 码是否正确（默认为 123456）
- 确认固件中的 `PASSKEY` 设置是否被修改
- 尝试重新配对：先断开连接，再重新连接
- 如果仍然失败，尝试重启 ESP32-C3

### Q3：命令发送后没有反应？

**检查清单**：
- 确认设备已连接（显示 "Connected" 状态）
- 检查命令是否正确（UNLOCK / LOCK / TRUNK）
- 确认光耦接线是否正确
- 用万用表测量 GPIO 引脚是否有输出
- 检查原车钥匙 PCB 是否正确焊接光耦输出端

### Q4：如何修改 PIN 码？

**步骤**：
1. 打开 Arduino IDE
2. 编辑 `firmware.ino` 中的 `#define PASSKEY 123456`
3. 改为你想要的 6 位数字（例如 654321）
4. 重新编译并上传固件到 ESP32-C3
5. 下次连接时使用新的 PIN 码

### Q5：设备连接后自动断开？

**可能原因**：
- 蓝牙信号干扰（远离金属或其他无线设备）
- 固件中的连接超时设置（默认 30 秒）
- iPhone 的蓝牙功耗管理
- 解决方案：靠近设备，尝试重新连接

## 🎯 典型使用流程

### 日常使用（每次 2-3 分钟）

1. **靠近车钥匙**（3-5 米范围内）
2. **打开 LightBlue App**
3. **点击 Scan 扫描设备**
4. **点击 "CarKey_BLE" 连接**
5. **输入 PIN 码**（如果首次连接）
6. **找到特征值 beb5483e-36e1-4688-b7f5-ea07361b26a8**
7. **输入命令**（UNLOCK / LOCK / TRUNK）
8. **点击 Write 发送**
9. **等待 1-2 秒，操作完成**
10. **点击 Disconnect 断开连接**

### 快速操作技巧

- **保持 App 打开**：在频繁操作时，保持 LightBlue 打开可以加快连接速度
- **收藏设备**：LightBlue 支持收藏常用设备，下次可以快速连接
- **预设命令**：某些 BLE 工具支持保存常用命令，可以一键发送

## 🔧 高级技巧

### 1. 批量操作


### 2. 自定义命令

如果你修改了固件，可以添加自定义命令。例如：
```
CUSTOM_CMD_1
CUSTOM_CMD_2
```

### 3. 监控信号强度

在 LightBlue 中，可以实时查看 RSSI（信号强度）：
- **-50 dBm**：非常强（靠近设备）
- **-60 dBm**：强（正常距离）
- **-70 dBm**：中等（较远距离）
- **-80 dBm**：弱（接近断开范围）

## 💡 最佳实践

1. **定期检查连接**：每次使用前确认设备在线
2. **及时断开**：操作完成后立即断开连接，节省电池
3. **保持距离**：在 3-5 米范围内使用，避免超出范围
4. **避免干扰**：远离金属、水和其他无线设备

## 📞 故障排查流程图

```
找不到设备？
  ├─ YES → 检查 ESP32-C3 是否上电
  │         ├─ 未上电 → 检查 CR2032 电池
  │         └─ 已上电 → 检查距离是否在 3-5 米内
  └─ NO → 继续

连接失败？
  ├─ YES → 检查 PIN 码是否正确
  │         ├─ 错误 → 输入正确 PIN 码
  │         └─ 正确 → 尝试重新配对
  └─ NO → 继续

命令无响应？
  ├─ YES → 检查光耦接线
  │         ├─ 接线错误 → 重新焊接
  │         └─ 接线正确 → 检查 GPIO 输出
  └─ NO → 操作成功！
```
